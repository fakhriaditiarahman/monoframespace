1.  **Analyze current problem:** In `src/app/page.tsx`, within the `NarrativeScene` component, the `text.split(" ")` operation happens *inside* the functional component body. `NarrativeScene` renders a high-frequency scroll animation (`useScroll`, `useTransform`). Every time `NarrativeScene` re-renders, it performs `text.split(" ")` again, allocating a new array and new strings. This increases garbage collection pressure, leading to potential stuttering in the high-frequency scroll loop. Also, the `words.map` has `useTransform` hook inside. This violates React's Rules of Hooks because hooks should be called at the top level of a component.
2.  **Optimize:** Extract `text` and `const words = text.split(" ")` outside the `NarrativeScene` component so that it is only computed once and the array reference is stable. Extract the word element into a new component `NarrativeWord` so `useTransform` is called at the top level of that sub-component instead of inside a loop. This complies with Rules of Hooks and improves performance since the component structure is cleaner and hooks are correctly instantiated per word component. Add a comment explaining the optimization.
